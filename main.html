Getting started

Connect your wallet (e.g. MetaMask) to read from and write to the Hariba contract. The contract is deployed at a fixed address; make sure you are on the correct chain. You need a small amount of native token (ETH or chain equivalent) for gas. To enqueue a task you also need to send the current fee in wei (see Dashboard).

Dashboard

Shows global counts (tasks, reminders, sessions, intents), pause status, and fee. If you are connected, it also shows your per-owner counts and balance.

Tasks

Lists the latest tasks in batch. Status: Pending (0), Completed (1), Cancelled (2). Kind: Generic (0), Call (1), Meeting (2), Deadline (3). To add a task, go to Actions and enqueue one; you must pay the current fee in wei.

Reminders

Reminders have a trigger timestamp and an optional linked task ID. Only the relay role can fire them on-chain when the trigger time has passed.

Sessions

Sessions are created by users and can receive response hashes (logged by the oracle) and slot data (curator). Close a session when done so no more responses are appended.

Intents

Intent types 0–7 are reserved for future use (e.g. NLU slot types). Registering an intent is a simple on-chain record.

Actions

From the Actions tab you can enqueue a task (with fee), set a reminder, create a session, complete or cancel a task, and close a session. All write operations require a connected wallet and sufficient gas.

Example workflow
  1. Connect your wallet (Connect button).
  2. Check Dashboard for fee and your balance.
  3. Go to Actions → Enqueue task: pick kind, set due timestamp (or use “Fill 1 day from now”), send tx with fee.
  4. Optionally set a reminder (trigger time, optional linked task ID).
  5. Create a session if you want to track conversation anchors.
  6. Later: complete or cancel a task from Actions (paste task ID).
  7. Close a session when done (paste session ID).

Timestamps

Use Unix timestamps (seconds since 1970-01-01). For example, one day from now: current time in seconds + 86400. The UI shows human-readable times where possible.

Data format

IDs (task, reminder, session, intent) are bytes32 hex (0x + 64 hex chars). Addresses are 0x + 40 hex (EIP-55 optional). Fee and balances are in wei (1 ether = 1e18 wei). Timestamps are Unix seconds (uint256).

Errors

Common contract errors: HRB_Paused (contract paused), HRB_ExceedsMaxTasksPerUser (user at cap), HRB_InsufficientDeposit (fee too low or balance too low), HRB_NotRelay (only relay can fire reminder), HRB_Unauthorized (not owner or keeper). Check your wallet and gas; ensure you are on the correct network.

FAQ

What is Hariba?

Hariba is a smart contract that stores tasks, reminders, sessions, and preferences on-chain. Cariboba is the web UI to read and write to it.

Do I need to pay to enqueue a task?

Yes. The contract charges a configurable fee in wei (set by the steward). Check the Dashboard for the current fee.

Who can fire a reminder?

Only the relay role can call fireReminder. Off-chain keepers watch trigger times and call the contract when due.

Security notes

Hariba uses reentrancy guards and role-based access. Only the vault can process withdrawals from the vault balance. Always verify the contract address and chain before sending transactions. Cariboba does not store your private key; signing is done in your wallet.

Browser support and privacy

Cariboba works in any modern browser that supports ES6 and has a Web3 wallet (e.g. MetaMask) or injects window.ethereum. No server round-trip for contract reads; RPC is used via the wallet or default provider. We do not collect or store your data; all state is on-chain or in your wallet.

Keyboard and accessibility

Use Tab to move between controls. Buttons and inputs have visible focus styles. All main actions are available from the tab panels; refresh the page to reload data or switch chain.

RPC and network

Reads use the provider from your wallet (e.g. MetaMask), so the RPC is whatever your wallet is connected to. If you see timeouts or empty data, try switching network or refreshing. Writes are sent through the same provider and require confirmation in the wallet.

Supported chains: any EVM chain where Hariba is deployed. The UI shows the current chain name and ID in the About tab after connection.

Feedback and intents

Submit feedback with a reference ID (bytes32) and rating 1–5. Register intents with type 0–7 for future NLU or slot classification. These are simple on-chain records; full payloads can live off-chain keyed by ref or intent ID.

Version and updates

Cariboba is a static single-file UI. To update the contract address or ABI, edit this HTML file. No backend or build step is required. Ethers.js is loaded from a CDN; for offline use, host the library locally. The Hariba contract is immutable after deploy; only config (fee, caps) and pause state can change.
